Indupalli Sumanth — Portfolio v0.1.0

Technical documentation for the personal portfolio website built with Next.js 14, TypeScript, Tailwind CSS, and Framer Motion.

This is a single-page portfolio application that showcases professional experience, projects, skills, education, and certifications. It features a dark/light theme toggle, scroll-triggered animations, a resume PDF download API, and a contact form.

Key Highlights Server-rendered with Next.js App Router • Fully responsive design • Dark/light theme with localStorage persistence • Animated splash screen • Server-side PDF resume generation via /api/resume • Scroll-reveal animations powered by Framer Motion • TypeScript strict mode throughout.

Architecture

The application follows the Next.js App Router pattern with a single-page layout. All sections are client-side React components loaded in the root page. The only server-side logic is the PDF generation API route.

Data Flow

  1. layout.tsx wraps the app in <ThemeProvider> and loads Google Fonts via CSS variables.
  2. page.tsx renders a splash screen with AnimatePresence, then reveals all section components.
  3. Each section component is a self-contained "use client" module with its own data (hardcoded arrays/objects).
  4. ScrollReveal wraps section content to trigger viewport-based entrance animations.
  5. The Navbar reads the theme context to render a dark/light toggle and tracks scroll position for active section highlighting.
  6. The /api/resume route generates a PDF on the server and returns it as a binary download.

Navbar

Fixed navigation bar at the top of the viewport.

Features

  • Scroll tracking — Uses a scroll event listener to detect which section is currently in view (by iterating nav links in reverse and checking getBoundingClientRect().top <= 150). The active link gets a pill-shaped highlight animated with Framer Motion's layoutId.
  • Glass morphism — After scrolling past 50px, applies a .glass (dark) or .glass-light (light) backdrop-blur class.
  • Theme toggle button — Renders a Sun/Moon icon that calls toggleTheme() from ThemeProvider.
  • Mobile drawer — On screens < 768px, a hamburger button toggles a slide-down menu animated with AnimatePresence.

ScrollReveal

A reusable wrapper component that animates its children into view when scrolled into the viewport.

Props

PropTypeDefaultDescription
childrenReactNode—Content to reveal
direction"up" | "down" | "left" | "right""up"Direction from which the content enters
delaynumber0Animation delay in seconds
width"fit-content" | "100%""100%"Container width
classNamestring""Additional CSS classes

Implementation

Uses Framer Motion's useInView hook (with once: true and margin: "-100px") to detect visibility, then triggers useAnimation to transition from hidden (60px offset, opacity 0) to visible (0 offset, opacity 1) over 0.6 seconds with an easeOut curve.

ThemeProvider

React Context provider managing dark/light theme state.

Context Value

interface ThemeContextType {
  theme: "dark" | "light";
  toggleTheme: () => void;
}

Behavior

  1. On mount, reads localStorage.getItem("theme"). Falls back to "dark".
  2. Adds/removes dark / light class on document.documentElement.
  3. Persists choice to localStorage on every change.
  4. Before hydration completes (mounted === false), renders children in a plain <div className="dark"> to prevent flash of unstyled content.

Consumed via the useTheme() hook in Navbar.tsx.

Theming (Dark / Light)

Theme switching is implemented via Tailwind's darkMode: "class" strategy.

How It Works

  1. ThemeProvider manages a "dark" | "light" state, persisted in localStorage.
  2. On state change, adds/removes .dark / .light on <html>.
  3. Components use Tailwind's dark: variant for conditional styles: dark:bg-dark-950 bg-white.
  4. CSS variables in globals.css change based on .light class (foreground, background, card-bg, etc.).
Hydration Note The initial HTML is always rendered with className="dark". On client mount, the stored theme is applied. suppressHydrationWarning is set on <html> to prevent React warnings during this swap.

Deployment

Vercel (Recommended)

Next.js is built by Vercel — deploy with zero configuration:

  1. Push the repository to GitHub.
  2. Import the project at vercel.com/new.
  3. Vercel auto-detects Next.js and configures build settings.
  4. Every push to main triggers a production deployment.

Static Export (Optional)

To export as a static site, set output: "export" in next.config.js. Note: the /api/resume route will not work in static mode as it requires a server runtime.
